# Устранение типичных ошибок

## «python» печатает только «Python» без версии / открывается Microsoft Store
В PATH стоит заглушка Store. Установите настоящий Python
(`winget install Python.Python.3.12`) или выключите заглушку:
Параметры → Приложения → «Псевдонимы выполнения приложения» → `python.exe` = выкл.
Используйте `.venv\Scripts\python` после создания окружения.

## `ModuleNotFoundError: No module named 'openpyxl'` (или yaml/rapidfuzz)
Не установлены зависимости или запущен не тот интерпретатор. Из корня проекта:
```
.venv\Scripts\python -m pip install -r requirements.txt
```
и запускайте скрипты через `.venv\Scripts\python`.

## «Не найден Excel контент-плана в inputs/current_cycle/»
Положите файл `.xlsx` контент-плана прямо в `inputs\current_cycle\`. Временные файлы
Excel вида `~$...xlsx` игнорируются — закройте файл в Excel перед запуском.

## Кириллица в CSV выглядит как «кракозябры»
Файлы пишутся в UTF-8 с BOM (Excel читает корректно). Если открываете сторонним
инструментом — выберите кодировку UTF-8. В самом Excel всё отображается верно.

## Roistat/Метрика «не подхватились»
Проверьте, что файлы лежат в `inputs\current_cycle\analytics\` и это исходные
выгрузки: Roistat — лист `Report` с колонкой «Посадочная страница»; Метрика — лист
`Отчет` со строкой «Отчет за период». Смотрите `01_input_audit.md` — там видно,
что распозналось.

## Один лист Excel «сломан» / необычные заголовки
Pipeline пропускает проблемный лист с предупреждением и продолжает. Список
предупреждений — в конце `01_input_audit.md`. Если пропущено важное — проверьте,
что на листе есть колонка «Тема» (для кейсов допустима «Описание»).

## План перекошен в один тип/кластер
Настройте `config/portfolio.yaml`: `strategy_mix` (доли ролей),
`balance.max_share_single_cluster` (потолок кластера). Изменения применяются при
следующем запуске.

## Слишком много/мало тем в неделю
`config/portfolio.yaml → capacity_defaults.items_per_week` или файл
`inputs/current_cycle/editorial_capacity.md` («N материалов в неделю»).
Лишние темы уходят в бэклог, а не ухудшают качество.

## Мои правки пропали после пересборки
Правки нужно вносить в `inputs/current_cycle/decisions.csv`, а не в Excel.
Файл Excel пересобирается заново каждый запуск.

## Ошибка при записи `content_plan.xlsx` (Permission denied)
Файл открыт в Excel. Закройте его и запустите снова.

## Хочу другой горизонт/старт цикла
```
.venv\Scripts\python scripts\run_pipeline.py --start 2026-08-04 --weeks 6
```
