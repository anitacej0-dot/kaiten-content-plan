# Ручные решения (decision log)

Главред управляет планом через один файл — `inputs/current_cycle/decisions.csv`.
Это **вход** pipeline: при каждом запуске решения применяются заново, поэтому
пересборка их не затирает. Шаблон — `inputs/templates/decisions.csv`.

## Формат

Колонки: `content_id, field, new_value, author, date, reason, is_exception`.

| Колонка | Значение |
|---|---|
| `content_id` | ID инициативы из плана (напр. `CP-202607-001`) |
| `field` | что менять: `decision_status` **или** любое поле инициативы |
| `new_value` | новое значение |
| `author` | кто принял решение |
| `date` | дата (YYYY-MM-DD) |
| `reason` | причина |
| `is_exception` | `да`, если это исключение из score |

## Примеры

Утвердить тему:
```
CP-202607-001,decision_status,утверждена,главред,2026-07-21,сильная тема с трафиком,нет
```
Отправить в бэклог:
```
CP-202607-050,decision_status,в бэклог,главред,2026-07-21,перенос на след. цикл,да
```
Поправить комментарий / автора / неделю:
```
CP-202607-002,editor_comment,добавить блок про интеграции,главред,2026-07-21,запрос продукта,нет
CP-202607-003,author,Егор,главред,2026-07-21,перераспределение,нет
```

## Значения `decision_status`

`утверждена` · `выбрана` · `в бэклог` · `отклонена` — перемещают тему между
разделами плана. `утверждена` также включает тему в handoff.

## Журнал

Все применённые решения фиксируются в `outputs/<цикл>/decision_log.csv` и на листе
«Decision log» в Excel: дата, кто, что изменено, было/стало, причина, исключение.

## Важно

- Не редактируйте `content_plan.xlsx` как источник истины — правки вносите в
  `decisions.csv`, иначе они пропадут при следующей пересборке.
- `content_id` стабильны в рамках одного набора входных данных. Если вы заменили
  Excel-план на новый — ID могут измениться; сверьтесь с новым `07_draft_master_plan.csv`.
