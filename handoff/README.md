# Передача в article pipeline

После утверждения плана создаётся `approved_topics.csv`.

Минимальные поля для каждой темы:
- content_id;
- тема;
- описание;
- тип материала;
- продуктовый кластер;
- сегмент и аудитория;
- интент;
- цель и ожидаемое действие;
- CTA / маршрут к продукту;
- ключевые доказательства;
- площадка;
- дедлайн;
- ссылка на существующие материалы;
- ограничения и комментарий главреда.

Article pipeline начинает работу только с утверждённых тем и не меняет стратегическую роль без возврата в decision log.
