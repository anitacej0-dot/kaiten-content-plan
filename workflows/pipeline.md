# Полный pipeline

Реализация — в пакете `pipeline/` и скриптах `scripts/`. Ниже — соответствие этапов
и артефактов в `outputs/YYYY-MM-DD_YYYY-MM-DD/`.

| Этап | Артефакт |
|---|---|
| Аудит входов | `01_input_audit.md` |
| Аудит истории | `02_history_audit.md` |
| Дубли/переносы/дистрибуция | `03_duplicates.csv` |
| Пул кандидатов | `04_candidates.csv` |
| Нормализация | `05_normalized_candidates.csv` |
| Оценка | `06_scored_candidates.csv` |
| Портфель | `07_draft_master_plan.csv` |

| Валидация | `10_validation_report.md` |
| Экспорт | `content_plan.xlsx`, `plan_summary.md`, `handoff/` |

## 0. Инициализация цикла

Создать папку `outputs/YYYY-MM-DD_YYYY-MM-DD/`, зафиксировать горизонт, мощность, продуктовые приоритеты и обязательные публикации.

## 1. Аудит истории

Проверить:
- повторяющиеся темы и переносы между месяцами;
- публикации, которые уже существуют в блоге или на внешних площадках;
- перекосы по продуктам, форматам и авторам;
- темы без описания, владельца, основания или следующего действия;
- сильные статьи, требующие Protect-обновления;
- каннибализацию и конкурирующие материалы.

Результат: `01_history_audit.md`.

## 2. Пул кандидатов

Сформировать кандидатов отдельно из SEO, аналитики, продаж/KAM, продуктовых запусков, кейсов, PR и редакционных инициатив. Не смешивать источники до фиксации доказательств.

Результат: `02_candidates.csv`.

## 3. Нормализация

Объединить дубли, связать новые темы с существующими URL, назначить продуктовый кластер, сегмент, воронку и стратегическую роль.

Результат: `03_normalized_candidates.csv`.

## 4. Оценка

Применить `config/scoring.yaml`. Для каждого балла оставить короткое человеческое обоснование. Автоматический балл не является финальным решением.

Результат: `04_scored_candidates.csv`.

## 5. Сборка портфеля

Сначала выбрать обязательные Protect/Convert/Prove-элементы, затем заполнить Build-темами. Проверить связки внутри кластеров и убрать темы, которые конкурируют между собой.

Результат: `05_draft_master_plan.csv`.

## 6. Внешняя дистрибуция

Для каждой подходящей инициативы решить:
- нужна ли внешняя площадка;
- какая именно и почему;
- будет ли адаптация, отдельный угол или новая статья;
- какое действие ожидается от читателя.

Результат: `06_placements.csv`.

## 7. Проверка мощности

Проверить количество материалов по неделям, авторов, экспертов, сложность, согласования и зависимые запуски. Перегруженные темы переносить в backlog, а не снижать качество.

Результат: `07_capacity_check.md`.

## 8. Ревью главреда

Главред проверяет логику, а не только формулировки. Все ручные изменения фиксируются в `decision_log.md`, чтобы pipeline учился на повторяющихся решениях.

## 9. Экспорт и handoff

Сформировать master plan, placements, summary и список утверждённых тем для article pipeline.
