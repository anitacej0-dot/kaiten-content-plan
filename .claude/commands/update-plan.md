# /update-plan

Пересобери план с учётом ручных решений главреда, не теряя их.

## Как это работает

Ручные решения хранятся в `inputs/current_cycle/decisions.csv` и являются входом:
при каждом запуске они применяются заново, поэтому пересборка их НЕ затирает.

Колонки: `content_id, field, new_value, author, date, reason, is_exception`.
- `field = decision_status`, `new_value = утверждена|выбрана|в бэклог|отклонена` —
  перемещает тему между разделами плана;
- любой другой `field` (`editor_comment`, `author`, `target_week`, `title`, …) —
  меняет значение поля инициативы.

## Шаги

1. Если главред продиктовал правки — внеси их строками в
   `inputs/current_cycle/decisions.csv` (создай файл из `inputs/templates/decisions.csv`,
   если его нет). Каждую правку оформляй отдельной строкой с причиной.
2. Перезапусти pipeline:
   ```
   .venv/Scripts/python scripts/run_pipeline.py
   ```
3. Сверься с `outputs/<цикл>/decision_log.csv` — там зафиксированы все применённые
   решения (что, кто, когда, было/стало, причина, исключение из score).
4. Кратко перескажи, что изменилось в плане (перемещения, новые статусы, правки полей),
   и обнови сводку из `plan_summary.md`.

Никогда не редактируй напрямую `content_plan.xlsx` как источник истины — правки
вносятся через `decisions.csv`, иначе они потеряются при следующей пересборке.
