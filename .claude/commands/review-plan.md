# /review-plan

Проведи редакционную проверку текущего чернового плана (без пересборки с нуля).

## Шаги

1. Убедись, что есть результаты последнего цикла в `outputs/<цикл>/`. Если нет —
   предложи сначала `/new-plan`.
2. При необходимости обнови отчёт валидации:
   ```
   .venv/Scripts/python scripts/validate_plan.py
   ```
3. Открой и разбери:
   - `10_validation_report.md` — дубли, роли, кластеры, сигналы, CTA, риски;
   - `09_capacity_check.md` — нагрузка по неделям и перегрузы;
   - `03_duplicates.csv` — переносы, дистрибуция, дубли, каннибализация;
   - `07_draft_master_plan.csv` и лист «Контент-план» в Excel.
4. Проверь по чек-листу и отметь проблемы:
   - у каждой темы есть роль, сигнал, CTA, продукт/кластер, сегмент, воронка;
   - нет тем-дублей без основания; дистрибуция не превращена в отдельные инициативы;
   - блог и внешние размещения связаны по `content_id`, у внешних есть свой угол;
   - баланс Protect/Convert/Build/Prove и кластеров в норме;
   - нет перегруженных недель и «одного продукта подряд»;
   - низкоприоритетные темы не попали в план без явного решения;
   - реалистичны авторы, эксперты и сроки.
5. Сформируй краткий отчёт по-русски: критические ошибки, спорные решения,
   рекомендуемые замены, готовность к утверждению. Предлагай конкретные правки
   в формате строк для `inputs/current_cycle/decisions.csv`.
