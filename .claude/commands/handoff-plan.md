# /handoff-plan

Собери пакеты утверждённых инициатив для передачи в `kaiten-article-pipeline`.

## Что делает

Для каждой инициативы со статусом `утверждена` создаётся YAML-файл в
`outputs/<цикл>/handoff/` со структурированным входом для article pipeline
(тема, цель, аудитория, продукт, проблема, роль, площадки, угол, CTA, сигналы,
доказательная база, ограничения, дедлайн, эксперты). Статью handoff НЕ пишет —
только готовит вход.

## Шаги

1. Убедись, что нужные темы утверждены. Если нет — добавь в
   `inputs/current_cycle/decisions.csv` строки вида:
   ```
   content_id,field,new_value,author,date,reason,is_exception
   CP-2026XX-001,decision_status,утверждена,главред,ГГГГ-ММ-ДД,причина,нет
   ```
2. Запусти экспорт (он же формирует handoff):
   ```
   .venv/Scripts/python scripts/export_plan.py
   ```
   Чтобы включить в handoff не только «утверждённые», но и все «выбранные» темы:
   ```
   .venv/Scripts/python scripts/export_plan.py --include-selected-in-handoff
   ```
3. Проверь `outputs/<цикл>/handoff/` — перечисли созданные пакеты и их `content_id`.
4. Сообщи главреду, сколько инициатив передано и какие остались неутверждёнными.

Если утверждённых тем нет — в каталоге будет только `README.md` с инструкцией, как
утвердить темы.
