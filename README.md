# Kaiten Content Plan Pipeline

Инструмент для подготовки контент-плана редакции Kaiten на **6–7 недель** вперёд.
Это **не** pipeline написания статей. Его задача — помочь главному редактору
**собрать, проверить, приоритизировать и оформить** контент-план для блога,
внешних площадок, кейсов, продуктовых и SEO-материалов, обновлений и исследований.

Главная единица работы — не статья и не размещение, а **контентная инициатива**:
тема с бизнес-целью, продуктовым кластером, сегментом, ролью в стратегии и
доказательством приоритета. Размещения (блог, Habr, VC, Бизнес-секреты, CRMindex и
др.) хранятся отдельно и связаны с инициативой по `content_id` — одна тема на блоге и
Habr это **одна инициатива с двумя размещениями**, а не два дубля.

## Что делает pipeline (одна команда)

1. Проверяет входные данные и делает их аудит.
2. Разбирает предыдущий Excel-план (десятки листов за разные месяцы и площадки).
3. Находит дубли, переносы, каннибализацию и дистрибуцию.
4. **Генерирует новые темы из данных** (спрос кластеров Roistat + паттерны редакции,
   дедуп с историей) и собирает пул кандидатов (новые идеи, переносы, Protect, кейсы, сигналы).
5. Нормализует темы, назначает кластер/роль/сегмент/воронку/тип.
6. Оценивает темы по стратегическим критериям (с обоснованием каждого балла).
7. Собирает сбалансированный портфель Protect/Convert/Build/Prove на 6–7 недель
   с учётом мощности редакции.
8. Планирует размещения на площадках.
9. Экспортирует готовый **Excel** (11 листов) и Markdown-сводку.
10. Готовит **handoff** утверждённых тем для `kaiten-article-pipeline`.

Ручные редакционные решения вносятся в один файл и **не теряются** при пересборке.

## Роли контента (стратегическая рамка)

- **Protect** — защитить работающий трафик и позиции (обновления, каннибализация).
- **Convert** — приблизить к заявке (сравнения, аналоги, внедрение, отраслевые сценарии).
- **Build** — строить новые кластеры (только при подтверждающем сигнале).
- **Prove** — доказать ценность (кейсы, исследования, экспертный контент).

Convert имеет повышенный вес, но портфель не забивается им механически — работают квоты.

## Быстрый старт для главреда (без навыков программирования)

### Первый запуск (один раз)

1. **Установите Python 3.11+**. Windows: в терминале
   ```powershell
   winget install Python.Python.3.12
   ```
   (или скачайте с python.org, отметив «Add python.exe to PATH»).
2. Откройте папку проекта в терминале и создайте окружение:
   ```powershell
   python -m venv .venv
   .venv\Scripts\python -m pip install -r requirements.txt
   ```

### Ежемесячный цикл

1. Положите данные в `inputs/current_cycle/` (подробно — `inputs/current_cycle/README.md`):
   - Excel предыдущего контент-плана — прямо в папку (файл не изменяется);
   - выгрузки Roistat и Метрики — в `inputs/current_cycle/analytics/`;
   - при желании — заполненные шаблоны сигналов из `inputs/templates/`.
2. Запустите одну команду:
   ```powershell
   .venv\Scripts\python scripts\run_pipeline.py
   ```
   Или в Claude Code — команду `/new-plan`.
3. Посмотрите результаты в `outputs/<дата>_<дата>/`:
   - `plan_summary.md` — сводка и вопросы к вам;
   - `content_plan.xlsx` — готовый план (открывается в Excel).
4. Внесите свои решения в `inputs/current_cycle/decisions.csv` и пересоберите:
   ```powershell
   .venv\Scripts\python scripts\run_pipeline.py
   ```
   Или `/update-plan`. Решения не теряются при повторном запуске.
5. Утверждённые темы передайте в article pipeline: `/handoff-plan`.

Если каких-то данных нет — pipeline не останавливается, а помечает пропуск и снижает
уверенность выводов (см. `01_input_audit.md` и `plan_summary.md`).

## Команды Claude Code

| Команда | Что делает |
|---|---|
| `/new-plan` | полный цикл: аудит → кандидаты → оценка → портфель → Excel + вопросы главреду |
| `/review-plan` | редакционная проверка чернового плана (дубли, баланс, риски, нагрузка) |
| `/update-plan` | пересобрать план с учётом решений из `decisions.csv`, не теряя правки |
| `/export-plan` | сформировать финальный Excel и сводку |
| `/handoff-plan` | собрать пакеты утверждённых инициатив для article pipeline |

## Итоговые артефакты цикла (`outputs/<дата>_<дата>/`)

```
01_input_audit.md          аудит входных данных
02_history_audit.md        аудит истории контента
03_duplicates.csv          дубли, переносы, дистрибуция, каннибализация
04_candidates.csv          сырой пул кандидатов
05_normalized_candidates.csv  нормализованные инициативы
06_scored_candidates.csv   оценка с обоснованием
07_draft_master_plan.csv   черновой план (одна строка = инициатива)
08_placements.csv          размещения по площадкам
09_capacity_check.md       проверка мощности по неделям
10_validation_report.md    отчёт валидации
plan_summary.md            сводка и вопросы главреду
decision_log.csv           журнал применённых ручных решений
content_plan.xlsx          ← ГОТОВЫЙ ФАЙЛ (11 листов)
handoff/                   пакеты для kaiten-article-pipeline
```
