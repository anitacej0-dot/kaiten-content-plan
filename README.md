# Kaiten Content Plan Pipeline

Репозиторий для подготовки контент-плана Kaiten на 6–7 недель для блога и внешних площадок.

Главная единица работы — не статья и не размещение, а **контентная инициатива**: тема с бизнес-целью, продуктовым кластером, сегментом, ролью в стратегии и доказательством приоритета. Размещения в блоге, Habr, VC, Бизнес-секретах и других площадках хранятся отдельно и связаны с инициативой по `content_id`.

## Что делает pipeline

1. Загружает стратегию, продуктовые приоритеты, данные SEO/аналитики, продажи, запуски и редакционные ограничения.
2. Проверяет предыдущие планы: дубли, переносы, каннибализацию, перекосы по кластерам и форматам.
3. Формирует единый пул кандидатов из разных источников.
4. Оценивает темы по влиянию на лидогенерацию и стратегию Protect → Convert → Build → Prove.
5. Собирает сбалансированный план на 6–7 недель с учётом мощности редакции.
6. Отдельно планирует внешнюю дистрибуцию и адаптации.
7. Выпускает итоговый план, сводку для согласования и handoff в article pipeline.

## Основные команды

- `/new-plan` — запустить новый цикл планирования.
- `/review-plan` — провести финальную редакционную проверку.

## Итоговые артефакты цикла

- `master_plan.csv` — одна строка на контентную инициативу.
- `placements.csv` — отдельные размещения и адаптации по площадкам.
- `plan_summary.md` — логика плана, баланс и ожидаемые действия.
- `rejected_topics.md` — темы, которые не вошли, с причиной.
- `handoff/approved_topics.csv` — темы для передачи в `kaiten-article-pipeline`.
